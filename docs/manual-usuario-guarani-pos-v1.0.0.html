1. Introducción

Guarani POS es un sistema de punto de venta pensado para operación diaria de comercios en Paraguay. Permite vender, controlar caja, administrar clientes y productos, registrar compras, llevar inventario, cobrar pagarés y consultar reportes operativos.

Importante: cada empresa visualiza solo su propia información. Los usuarios, productos, clientes, ventas, compras y reportes quedan separados por empresa.

2. Acceso al sistema

  1. Ingresar a la pantalla de login.
  2. Completar empresa, usuario y contraseña.
  3. Presionar Ingresar.
  4. Si el sistema lo permite, también se puede usar acceso por PIN.

Los permisos visibles dependen del rol del usuario: administrador, supervisor o cajero.

3. Dashboard

El dashboard resume la operación actual de la empresa.

Indicadores principales

  • Ventas del día
  • Ventas del mes
  • Productos con bajo stock
  • Presupuestos pendientes
  • Pagarés vencidos
  • Usuarios activos

Uso recomendado

  • Revisar rápidamente el estado operativo al comenzar el día
  • Controlar alertas de stock bajo
  • Detectar pendientes comerciales o financieros

4. Caja

La caja controla la apertura, movimientos y cierre del turno.

FunciónDescripción
Abrir cajaPermite iniciar el turno con un monto de apertura y observación opcional.
Ingresos y retirosPermite registrar movimientos manuales durante el día.
Cierre de cajaCompara el efectivo esperado con el contado y calcula diferencias.
HistorialMuestra las cajas abiertas y cerradas, con opción de reporte y PDF.
Importante: si existe diferencia al cerrar la caja, el sistema exige observación.

5. POS

El módulo principal del sistema permite registrar ventas de forma rápida.

Funciones principales

  • Búsqueda de productos por código o nombre
  • Búsqueda y alta rápida de clientes
  • Descuento por ítem
  • Descuento global
  • Venta con múltiples medios de cobro
  • Suspender y reanudar venta
  • Vista previa, reimpresión y PDF

Reglas destacadas

  • Se requiere caja abierta para vender
  • El precio mayorista se aplica automáticamente por cantidad cuando corresponde
  • Los límites de descuento dependen del rol y de parámetros configurados
  • Las devoluciones y anulaciones ajustan stock y caja

6. Presupuestos

  • Permite crear presupuestos con cliente, vigencia y observación.
  • Soporta descuento por línea y descuento global.
  • Permite aprobación, rechazo y marcado como enviado.
  • Incluye vista previa y PDF del presupuesto.
  • Puede convertirse a venta cuando corresponda.

7. Clientes

Desde esta pantalla se administran los datos del cliente.

  • Nombre, documento, RUC, teléfono, correo y dirección
  • Tipo de documento
  • Segmento comercial: minorista, mayorista u otros
  • Perfil fiscal: contribuyente, consumidor final u otros
  • Observación y estado activo/inactivo

8. Productos

  • Alta y edición de productos
  • Precio costo
  • Precio de venta 1
  • Precio de venta 2 para mayorista
  • Cantidad mínima para activar precio mayorista
  • Stock actual y stock mínimo
  • Importación masiva desde Excel

9. Inventario

  • Resumen de stock actual
  • Filtro de productos con stock bajo
  • Ajuste manual de stock con motivo
  • Kardex rápido por producto
  • Movimientos por venta, devolución, anulación y compra
  • Exportación de inventario y kardex

10. Compras

Operación

  • Alta de proveedores
  • Compra directa
  • Orden de compra
  • Recepción parcial o total
  • Pagos al proveedor

Controles

  • La compra impacta inventario
  • Pago en efectivo genera retiro en caja
  • Si no hay caja abierta o no alcanza el efectivo, se bloquea
  • Se controla saldo pendiente y estado de pago

11. Pagarés

  • Alta de pagarés manuales o vinculados a venta
  • Pago parcial o total
  • Control de saldo pendiente y vencimiento
  • Si el pago es en efectivo, genera ingreso en caja
  • Auditoría específica de pagarés

12. Reportes

La pantalla de reportes permite consultar el período elegido y exportar resultados.

  • Ventas por día
  • Ventas por cajero
  • Productos más vendidos
  • Caja diaria
  • Anulaciones y devoluciones
  • Exportación en Excel y PDF

14. Buenas prácticas de uso

  1. Abrir caja antes de comenzar a vender o cobrar.
  2. Registrar siempre ingresos y retiros manuales en el momento.
  3. Revisar stock bajo de forma diaria.
  4. Usar clientes correctamente identificados cuando se requiere comprobante fiscal.
  5. Verificar el cierre de caja todos los días.
  6. Respaldar la base de datos con frecuencia.
Implementaciones futuras: el sistema deja preparado el camino para integraciones posteriores como Bancard y otras mejoras avanzadas, pero este manual describe únicamente las funciones disponibles en la versión 1.0.0.
